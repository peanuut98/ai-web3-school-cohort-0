# Handbook Feedback

> 我在读 Handbook 时遇到的卡点、错别字、概念不清、资料过期、结构建议都放这里。

## 文件命名

`YYYY-MM-DD-<short-slug>.md`，例如 `2026-05-15-wallet-derivation-path.md`。

## 每条 feedback 至少包含

1. **Handbook 页面链接**（精确到章节锚点）
2. **问题描述**（看到这一段我卡住了 / 我以为是 X 但实际是 Y）
3. **建议改法**（如果是错别字直接给原文+建议；如果是结构问题给一句话提案）
4. **来源**（哪一天 daily note 里第一次出现的）

## 模板

```markdown
# {{Slug}}

- 页面：https://aiweb3.school/zh/handbook/...
- 类型：错别字 / 概念不清 / 资料过期 / 结构建议 / 其他
- 来源 daily：daily/YYYY-MM-DD.md

## 问题
（一两句话说清楚）

## 建议改法
（具体到改哪句、加哪段、调哪个顺序）

## 补充上下文
（可选：我当时在做什么，因此踩到这个坑）
```

## 提交节奏

- 每周五至少 1 条
- 攒到 3–5 条时，可以考虑去 Handbook 仓库提 issue / PR（保留这里作为底稿）
