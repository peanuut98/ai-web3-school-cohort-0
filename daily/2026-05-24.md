## 🔁 Base Sepolia 测试网交易复盘

整个流程拆成 4 段：

### 1. 准备

- 钱包：Rabby Wallet（浏览器插件）
- 网络：Base Sepolia（不是 Ethereum Sepolia — Base 是建在 Ethereum 上的 L2）
- 测试 ETH 来源：Base Sepolia faucet
- From：我自己的测试地址 A
- To：我自己另一个测试地址 B（同一个钱包里的另一个账户，纯测试）

### 2. 在 Rabby 里发起转账

- 选中 From 地址，点 Send，粘贴 To 地址，输入金额。
- Rabby 弹出确认页，里面显示了：To / Amount / 网络 / 预估 Gas fee。
- 这一步对应学过的「签名」 — 我点 Confirm 时，Rabby 用 A 的私钥对这条 tx 数据签了名，然后广播到 Base Sepolia 节点。

### 3. 在 BaseScan 上找回这笔交易

- 打开 https://sepolia.basescan.org （Base Sepolia 的区块浏览器）
- 把地址 A 粘到搜索框，最近一笔 tx 就是它。
- 点进去看到 7 个字段都符合预期：

| 字段 | 含义 | 验证结果 |
| --- | --- | --- |
| Status | Success / Pending / Failed | Success |
| Tx Hash | 这笔交易的唯一指纹 | 0x... 64 hex 字符，唯一 |
| Block | 被打包进的区块号 | 一个具体数字，能点进去看这个区块的所有交易 |
| Timestamp | 上链时间 | 和我点 Confirm 的时间差几秒，符合预期 |
| From | 发送方 | 我自己地址 A |
| To | 接收方 | 我自己地址 B |
| Value | 随交易转的 ETH | 我设置的金额（Base Sepolia 测试 ETH，无价值） |
| Txn Fee | 实际付的 Gas 费 | 一个很小的数，B 没收到这部分，是给网络的 |

### 4. 关键确认

- Tx Hash 在 BaseScan 上能搜到 → 这笔交易确实上链了，**不只是「钱包显示成功」**。
- B 的余额增加 = Value，From 的余额减少 = Value + Txn Fee — 数字对得上。
- 这一步是分清「钱包侧成功」和「链上侧成功」的关键。

## 🧠 我对几个核心概念的理解（用今天这笔交易解释）

- **交易哈希（Tx Hash）**：一笔 tx 的全网唯一指纹。把整个 tx 内容做 hash 得到，**不是钱包发的，是数据本身决定的**。所以同样内容只能算出同一个 hash，篡改任何一字节 hash 就完全不一样（Day 2 用 hello / Hello 的 SHA-256 已经直观验证过）。
- **区块浏览器（BaseScan / Etherscan）**：链是公开账本，浏览器是这个账本的搜索引擎。它本身**不验证我**，只是把链上数据用人能读的方式展示。这也意味着：浏览器查不到的 tx 等于没上链。
- **Gas / Txn Fee**：我付给打包节点的钱，不是给收款方的。`Txn Fee = Gas Used × Gas Price`。本次 Txn Fee 极低，因为 Base 是 L2，且我做的是最简单的 ETH 转账（Gas Used 接近 21000）。
- **From / To**：From 是「谁签了名」，To 是「这条 tx 指向谁 / 哪个合约」。两者都长一样（0x... 42 字符），靠 BaseScan 的 Contract 标记区分钱包地址和合约地址。我这次 To 没有 Contract 标记，是普通钱包地址。

一个之前没想清楚、今天想明白的点：**Gas 不在 Value 里**。Value 是 To 收到的，Gas 是网络拿走的，From 余额扣掉的总数 = Value + Txn Fee。这个公式以前看过，但今天才在自己的余额变化上对上号。

## 🤔 哪些步骤必须人工确认（再快也不能跳）

1. **网络是不是 Base Sepolia**（不是主网） — 测试操作前先看 Rabby 顶部的网络标签。
2. **To 地址的前 6 位 + 后 6 位** — 防止剪贴板劫持。中间字符肉眼通常不看，但首尾必须比对。
3. **Value 的金额位数** — 多一个 0 就是 10 倍，wei 视图下尤其容易看错。
4. **Gas fee 量级是否合理** — 一笔普通 ETH 转账 Txn Fee 应该是「极小」级别，如果突然要 0.x ETH 那一定是哪里不对。
5. **Tx Hash 在 BaseScan 上能搜到** — 这是「钱包成功」之外的二次验证。
6. **签名内容（如果是签消息而不是转账）** — 看不懂的 typed data，不签。

钱包替我做的：
- 用私钥生成签名
- 估算 Gas
- 把 tx 广播到节点
- 监听上链状态并更新 UI

但**网络选择 / 地址核对 / 金额核对 / 内容理解**这 4 件事钱包帮不了我，必须人工。

## ❓ 遇到的问题和解决方式

- **问题 1：Rabby 顶部默认网络不是 Base Sepolia，差点在错误网络上发。**
  - 解决：每次 Send 前先看一眼网络下拉，养成肌肉记忆。
- **问题 2：第一次找 Base Sepolia faucet 时点到了一个伪装的网站。**
  - 解决：只通过官方 docs / Base 官网链接进 faucet，不在搜索引擎广告位点。
- **问题 3：在 BaseScan 上同一个地址有多笔 tx，分不清哪笔是我刚发的。**
  - 解决：按 Timestamp 排序，最新一笔；同时核对 Tx Hash 前几位（钱包里有显示）。
- **问题 4：以为钱包余额变化 = Value，结果 From 多扣了一点。**
  - 解决：把 Txn Fee 单列，公式 `From 扣减 = Value + Txn Fee` 写到笔记里。
