## 关键概念笔记 · Part 1：ethers.js 最小知识（够跑今天这段脚本）

今天不是系统学 ethers.js，是**「够用就行」** — 写下跑通这段脚本需要的最小知识，免得被文档淹没。

### ethers.js 三个核心对象（今天用到的）

| 对象 | 一句话 | 今天怎么用 |
| --- | --- | --- |
| **Provider** | 连接到以太坊网络的入口 — 负责发 RPC 请求 | `new ethers.JsonRpcProvider(rpcUrl)` 连 Sepolia |
| **Wallet** | 持有私钥、能签名的账户对象 | 今天**不用** — 只读余额不需要签名 |
| **Contract** | 代表一个已部署的智能合约，能调它的方法 | 今天**不用** — 读余额用 provider 就够 |

今天的脚本只用到 **Provider**，是 ethers.js 里最轻的一档。

### 读余额的最小代码（4 行）

```javascript
import { ethers } from 'ethers';

const provider = new ethers.JsonRpcProvider(process.env.SEPOLIA_RPC_URL);
const address = '0x...';  // 目标地址
const balance = await provider.getBalance(address);
console.log(ethers.formatEther(balance));  // Wei → ETH
```

关键点：
- `getBalance` 返回的是 **Wei**（1 ETH = 10^18 Wei）— 必须用 `formatEther` 转成人类可读的 ETH。
- RPC URL 从环境变量读，**不硬编码** — 这是 Vibe Coding 的硬要求（Day 15 强调过「不把 secret 写进代码」）。
